## Отчет по Заданию №1: Настройка программного обеспечения для анализа больших данных

### Наименование работы
Настройка программного обеспечения для анализа больших данных

### Цель работы
Научиться выбирать, устанавливать и настраивать прикладное программное обеспечение, соответствующее конкретным задачам профессиональной деятельности, а также обеспечивать совместимость, безопасность и функциональность программ в рамках заданной операционной системы и инфраструктуры.

### Задание
Для обеспечения работы отдела аналитики необходимо установить и настроить программное обеспечение, подходящее для анализа больших данных. Вам потребуется:
▪ Установить платформу для обработки и анализа больших объемов данных, поддерживающую распределенную обработку.
▪ Подключить визуализационное ПО для построения графиков и диаграмм.
▪ Установить клиент для работы с SQL-серверами.
▪ Убедиться, что используемые программы соответствуют требованиям безопасности и совместимы с ОС Windows.

### Сравнение систем для анализа данных

Ниже представлен список программного обеспечения, широко используемого для анализа больших данных, с кратким описанием ключевых особенностей каждого:

1.  **Apache Hadoop**
    *   **Назначение:** Фреймворк с открытым исходным кодом для распределенного хранения и обработки очень больших наборов данных в кластерах компьютеров.
    *   **Ключевые особенности:**
        *   **HDFS (Hadoop Distributed File System):** Распределенная файловая система для хранения огромных объемов данных.
        *   **MapReduce:** Модель программирования для параллельной обработки больших наборов данных.
        *   **Экономичность:** Позволяет использовать недорогое стандартное оборудование.
        *   **Масштабируемость:** Легко масштабируется для обработки растущих объемов данных.
        *   **Надежность:** Разработан для обнаружения и обработки сбоев.
    *   **Типичные сценарии:** Хранение и архивирование больших данных, пакетная обработка, ETL-процессы.

2.  **Apache Spark**
    *   **Назначение:** Унифицированный движок для крупномасштабной обработки данных, который может работать как самостоятельно, так и поверх Hadoop. Отличается высокой скоростью благодаря обработке данных в оперативной памяти.
    *   **Ключевые особенности:**
        *   **Скорость:** В 10-100 раз быстрее, чем Hadoop MapReduce, за счет обработки в памяти.
        *   **Унифицированный движок:** Поддерживает пакетную обработку, интерактивные запросы, потоковую передачу данных, машинное обучение и графовую обработку.
        *   **MLlib:** Встроенная библиотека для машинного обучения.
        *   **Spark Streaming:** Для обработки данных в реальном времени.
        *   **Поддержка языков:** API для Scala, Python (PySpark), Java и R.
    *   **Типичные сценарии:** Интерактивный анализ данных, машинное обучение, потоковая обработка, анализ клиентских жалоб, обнаружение мошенничества.

3.  **Microsoft SQL Server (с акцентом на SQL Server 2022)**
    *   **Назначение:** Мощная реляционная система управления базами данных (РСУБД) с расширенными возможностями для бизнес-анализа, интеграции данных и машинного обучения.
    *   **Ключевые особенности:**
        *   **Интеграция:** Глубокая интеграция с экосистемой Microsoft (Windows, Azure, Power BI).
        *   **Производительность:** Оптимизирован для высокой производительности и масштабируемости, включая Intelligent Query Processing.
        *   **Расширенная аналитика:** Встроенные службы машинного обучения (R и Python) и Analysis Services (SSAS).
        *   **Безопасность:** Многоуровневая защита данных и инновационные функции безопасности.
        *   **Гибридные возможности:** Поддержка локальных и облачных развертываний, а также Windows, Linux и Docker-контейнеров.
        *   **Аналитика в реальном времени:** Возможности для операционной аналитики с минимальным влиянием на производительность.
    *   **Типичные сценарии:** Традиционные хранилища данных, операционная аналитика, бизнес-аналитика в реальном времени, приложения, требующие высокой производительности и безопасности.

4.  **NoSQL базы данных (например, MongoDB, Cassandra)**
    *   **Назначение:** Не реляционные базы данных, предназначенные для хранения и обработки неструктурированных или полуструктурированных данных, а также для горизонтального масштабирования.
    *   **Ключевые особенности:**
        *   **Гибкая схема:** Позволяют хранить данные без строгой предопределенной схемы.
        *   **Горизонтальное масштабирование:** Легко распределяются по множеству серверов.
        *   **Высокая доступность:** Многие NoSQL базы данных обеспечивают высокую доступность и отказоустойчивость.
        *   **Различные модели данных:** Документоориентированные, колоночные, графовые, ключ-значение.
    *   **Типичные сценарии:** Хранение пользовательских профилей, каталогов продуктов, данных IoT, социальных сетей, больших объемов логов.

5.  **Облачные платформы для больших данных (например, Google Cloud BigQuery, Amazon Redshift, Azure Synapse Analytics)**
    *   **Назначение:** Полностью управляемые облачные сервисы, предоставляющие инфраструктуру и инструменты для хранения, обработки и анализа больших данных без необходимости управления серверами.
    *   **Ключевые особенности:**
        *   **Полностью управляемые:** Снимают нагрузку по управлению инфраструктурой.
        *   **Масштабируемость по требованию:** Автоматическое масштабирование ресурсов.
        *   **Интеграция:** Тесная интеграция с другими облачными сервисами и инструментами.
        *   **Оплата по мере использования:** Гибкая модель ценообразования.
    *   **Типичные сценарии:** Построение облачных хранилищ данных, аналитика в реальном времени, машинное обучение в облаке, обработка потоковых данных.

Этот список охватывает основные категории программного обеспечения, используемого в современном анализе больших данных, от фреймворков с открытым исходным кодом до коммерческих СУБД и облачных решений.

# Отчет по выполнению Задания 2: Управление файловой системой

В данном отчете описывается последовательность шагов для выполнения Задания 2.

## 1. Создание и настройка дисковых разделов

Для создания новых разделов диска мы будем использовать PowerShell. Так как на диске D нет важных данных, мы можем его переиспользовать.

1.  **Удаление существующего раздела:** Сначала мы удалим раздел D, чтобы освободить место.
2.  **Создание новых разделов:** Затем мы создадим два новых раздела на освободившемся пространстве.
3.  **Форматирование разделов:** Мы отформатируем новые разделы с файловой системой NTFS и присвоим им метки.

**Важно:** Для выполнения следующих команд требуются права администратора. Откройте PowerShell от имени администратора и выполните следующие команды.

### Удаление раздела

```powershell
Remove-Partition -DiskNumber 0 -PartitionNumber 5 -Confirm:$false
```

После выполнения этой команды раздел D будет удален. Далее мы создадим новые разделы.

### Создание и форматирование новых разделов

```powershell
# Создаем первый раздел размером 10ГБ
New-Partition -DiskNumber 0 -Size 10GB -AssignDriveLetter | Format-Volume -FileSystem NTFS -NewFileSystemLabel "Data1"

# Создаем второй раздел размером 10ГБ
New-Partition -DiskNumber 0 -Size 10GB -AssignDriveLetter | Format-Volume -FileSystem NTFS -NewFileSystemLabel "Data2"
```

Эти команды создадут два новых раздела, отформатируют их в файловой системе NTFS и присвоят им метки "Data1" и "Data2".

## 2. Настройка прав доступа к папкам

В этом разделе мы создадим двух пользователей и настроим права доступа к папке таким образом, чтобы один пользователь имел доступ, а другой — нет.

1.  **Создание пользователей:** Мы создадим двух локальных пользователей, `User1` и `User2`.
2.  **Создание папки:** Мы создадим папку, к которой будем настраивать доступ.
**Важно:** Для выполнения следующих команд требуются права администратора. Откройте PowerShell от имени администратора и выполните следующие команды.

### Создание пользователей

```powershell
# Задаем пароль для новых пользователей
$Password = ConvertTo-SecureString "Pa$w0rd123" -AsPlainText -Force

# Создаем пользователей
New-LocalUser "User1" -Password $Password -FullName "User One" -Description "Test user 1"
New-LocalUser "User2" -Password $Password -FullName "User Two" -Description "Test user 2"
```

### Создание папки и настройка прав

```powershell
# Создаем папку
New-Item -Path "C:\Temp\SecretFolder" -ItemType Directory

# Предоставляем полный доступ для User1
icacls "C:\Temp\SecretFolder" /grant "User1:(OI)(CI)F"

# Запрещаем доступ для User2
icacls "C:\Temp\SecretFolder" /deny "User2:(OI)(CI)F"
```

Эти команды создадут двух пользователей, папку, и настроят права доступа к ней. `(OI)(CI)F` означает, что права наследуются для всех вложенных файлов и папок, и `F` предоставляет полный доступ.

## 3. Шифрование папки с помощью BitLocker

BitLocker — это технология шифрования дисков, которая позволяет зашифровать целый том (диск). Нельзя зашифровать отдельную папку с помощью BitLocker, но можно зашифровать том, на котором находится эта папка.

В этом разделе мы зашифруем один из созданных нами ранее дисков (например, тот, что с меткой "Data1").

**Важно:** Для выполнения следующих команд требуются права администратора. Откройте PowerShell от имени администратора и выполните следующие команды.

### Включение BitLocker

Сначала определите букву диска, который вы хотите зашифровать (например, E:).

```powershell
# Включить BitLocker для диска E: с паролем
# Замените "E:" на букву вашего диска
Enable-BitLocker -MountPoint "E:" -EncryptionMethod XtsAes256 -PasswordProtector
```

После выполнения этой команды система попросит вас ввести пароль для диска. Этот пароль нужно будет вводить каждый раз при доступе к диску.

**Важно:** Сохраните ключ восстановления, который будет сгенерирован. Он понадобится для доступа к данным в случае утери пароля.

## 4. Подключение сетевой папки в качестве сетевого диска

В этом разделе мы создадим общую сетевую папку и подключим ее как сетевой диск.

1.  **Создание общей папки:** Мы создадим папку и сделаем ее доступной по сети.
**Важно:** Для выполнения следующих команд требуются права администратора. Откройте PowerShell от имени администратора и выполните следующие команды.

### Создание общей папки

```powershell
# Создаем папку для общего доступа
New-Item -Path "C:\Temp\SharedFolder" -ItemType Directory

# Открываем к ней доступ по сети
New-SmbShare -Name "SharedFolder" -Path "C:\Temp\SharedFolder" -FullAccess "Everyone"
```

### Подключение сетевого диска

```powershell
# Подключаем сетевую папку как диск Z:
New-PSDrive -Name "Z" -PSProvider "FileSystem" -Root "\\localhost\SharedFolder" -Persist
```

Эта команда создаст новый диск Z:, который будет указывать на сетевую папку `\\localhost\SharedFolder`. Ключ `-Persist` делает диск доступным после перезагрузки.

## 5. Создание архива данных с паролем

Важно отметить, что встроенными средствами Windows невозможно создать архив, защищенный паролем. Для этой цели необходимо использовать сторонние утилиты. Мы воспользуемся сторонней утилитой 7-Zip, так как встроенные средства Windows не поддерживают установку пароля на архив через командную строку.

1.  **Установка 7-Zip:** Если 7-Zip не установлен, его необходимо скачать и установить.
2.  **Создание архива:** Мы используем команду 7-Zip для создания архива с паролем.

### Установка 7-Zip

Скачайте и установите 7-Zip с официального сайта: [https://www.7-zip.org/](https://www.7-zip.org/)

### Создание архива

После установки 7-Zip, вы можете использовать следующую команду в PowerShell или CMD.

```powershell
# Путь к исполняемому файлу 7-Zip
$sevenZipPath = "C:\Program Files\7-Zip\7z.exe"

# Папка, которую нужно заархивировать
$folderToArchive = "C:\Temp\SecretFolder"

# Имя архива
$archiveName = "C:\Temp\SecretArchive.7z"

# Пароль
$password = "SuperSecretPassword"

# Команда для создания архива
& $sevenZipPath a -p"$password" "$archiveName" "$folderToArchive"
```

Эта команда создаст архив `SecretArchive.7z` в папке `C:\Temp`, защищенный паролем.

## 1. Назначение сервиса

**Gitea** — это легковесный веб-сервис для хостинга IT-проектов и совместной разработки, написанный на языке Go. Он предоставляет систему управления версиями Git, а также включает в себя инструменты для отслеживания ошибок, вики и просмотра кода. Gitea является аналогом таких сервисов, как GitHub, Bitbucket или GitLab, но предназначен для самостоятельного хостинга на собственном оборудовании.

Данное руководство описывает работу с локально установленной версией Gitea, запущенной с помощью Docker.

## 2. Установка и запуск

Для запуска Gitea на вашем компьютере используется технология контейнеризации Docker.

1.  **Установите Docker Desktop** с официального сайта [docker.com](https://www.docker.com/products/docker-desktop/).
2.  Откройте терминал (командную строку PowerShell или CMD в Windows).
3.  Выполните следующую команду для скачивания и запуска контейнера Gitea:

    ```bash
    docker run -d --name=gitea -p 3000:3000 -p 2222:22 -v gitea:/data gitea/gitea:latest
    ```
    *   `-d` — запуск в фоновом режиме.
    *   `--name=gitea` — имя контейнера.
    *   `-p 3000:3000` — проброс веб-порта Gitea на порт `3000` вашего компьютера.
    *   `-p 2222:22` — проброс SSH-порта на порт `2222` (чтобы не конфликтовать с системным SSH).
    *   `-v gitea:/data` — создание и монтирование тома `gitea` для хранения всех данных (репозиториев, настроек, БД).

После выполнения команды сервис Gitea будет доступен в браузере по адресу `http://localhost:3000`.

## 3. Требования к системе и ПО

*   **Операционная система:** Windows 10/11, macOS или Linux с поддержкой Docker.
*   **Программное обеспечение:**
    *   Установленный **Docker Desktop** или **Docker Engine**.
    *   Современный веб-браузер (Google Chrome, Mozilla Firefox, Microsoft Edge).
    *   Установленный клиент **Git** для работы с репозиториями из командной строки.
*   **Интернет-соединение:** Требуется для первоначального скачивания Docker-образа Gitea. Для дальнейшей работы с локальным сервисом не обязательно.

## 4. Первоначальная настройка и регистрация

### 4.1. Настройка Gitea

При первом входе на `http://localhost:3000` откроется страница первоначальной настройки.

1.  **Тип базы данных:** Оставьте `SQLite3`, это самый простой вариант для локального использования.
2.  **Общие настройки:**
    *   **Заголовок сайта:** Введите название вашего сервиса, например, "Локальный Git".
    *   **Базовый URL-адрес сервера:** Должен быть `http://localhost:3000/`.
    *   **URL-адрес для клонирования по SSH:** Измените порт на `2222`, чтобы получилось `ssh://git@localhost:2222/`.
3.  **Дополнительные настройки:**
    *   Вы можете отключить самостоятельную регистрацию пользователей (`Отключить регистрацию`), чтобы только администратор мог создавать аккаунты.
    *   Создайте **учетную запись администратора** прямо в этой форме, задав логин и пароль.
4.  Нажмите кнопку **"Установить Gitea"**.

*(Подсказка: здесь можно сделать скриншот страницы первоначальной настройки Gitea)*

### 4.2. Регистрация и авторизация

*   **Авторизация:** После установки вы будете перенаправлены на страницу входа. Используйте логин и пароль администратора, которые вы указали на предыдущем шаге.
*   **Регистрация:** Если вы не отключали регистрацию, новые пользователи могут создать аккаунт, нажав на ссылку «Зарегистрироваться» на главной странице.

## 5. Использование основных функций

### 5.1. Создание репозитория

1.  Нажмите на значок `+` в правом верхнем углу и выберите **«Новый репозиторий»**.
2.  Введите **имя репозитория**, добавьте описание.
3.  Вы можете сделать репозиторий **приватным** (доступен только вам и тем, кому вы дадите права) или **публичным**.
4.  Нажмите **«Создать репозиторий»**.

*(Подсказка: здесь можно сделать скриншот страницы создания нового репозитория)*

### 5.2. Клонирование репозитория

После создания репозитория вы увидите его URL-адреса (HTTPS и SSH). Чтобы скопировать файлы на свой компьютер, используйте команду `git clone` в терминале.

**Пример для HTTPS:**
```bash
git clone http://localhost:3000/имя_админа/имя_репозитория.git
```
При выполнении `push` или `pull` потребуется вводить логин и пароль от Gitea.

**Пример для SSH (рекомендуется):**
Для этого сначала нужно добавить ваш публичный SSH-ключ в настройки профиля Gitea.
```bash
git clone ssh://git@localhost:2222/имя_админа/имя_репозитория.git
```

### 5.3. Работа с репозиторием: commit и push

1.  Перейдите в папку с клонированным репозиторием.
2.  Внесите изменения в файлы, создайте новые.
3.  Добавьте файлы в индекс: `git add .`
4.  Зафиксируйте изменения (сделайте коммит): `git commit -m "Мой первый коммит"`
5.  Отправьте изменения на сервер Gitea: `git push origin main`

### 5.4. Создание веток и Pull Request'ов

Для совместной работы принято создавать изменения в отдельных ветках и предлагать их к слиянию через Pull Request.

1.  Создайте новую ветку: `git checkout -b new-feature`
2.  Сделайте коммиты в новой ветке и отправьте ее на сервер: `git push origin new-feature`
3.  В интерфейсе Gitea появится предложение создать **Pull Request**. Нажмите на него, опишите ваши изменения и создайте запрос.
4.  Другой участник проекта (или вы сами) может просмотреть изменения, оставить комментарии и, если все в порядке, **объединить (merge)** их с основной веткой.

## 6. Рекомендации по безопасности

*   **Используйте SSH-ключи** для доступа к репозиториям вместо паролей. Это удобнее и безопаснее.
*   Назначайте **сложный пароль** для аккаунта администратора.
*   **Управляйте доступом** к репозиториям: для приватных проектов добавляйте соавторов (collaborators) с определенными правами (чтение, запись).
*   **Не храните** в репозиториях пароли, ключи API и другие секретные данные в открытом виде.

## 7. Устранение неполадок и восстановление доступа

*   **Сервис недоступен по `http://localhost:3000`:**
    1.  Проверьте, запущен ли Docker Desktop.
    2.  Выполните в терминале команду `docker ps` и убедитесь, что контейнер `gitea` есть в списке и имеет статус `Up`.
    3.  Если контейнера нет, запустите его командой `docker start gitea`.
*   **Ошибка аутентификации при `git push`:**
    1.  Убедитесь, что вы правильно вводите логин и пароль (для HTTPS).
    2.  Проверьте, что ваш SSH-ключ добавлен в Gitea и используется правильный URL (для SSH).
*   **Восстановление пароля:** На странице входа есть ссылка «Забыли пароль?». Gitea отправит инструкции по сбросу на e-mail, указанный в профиле. Для локальной установки без настроенного почтового сервера эта функция работать не будет. В таком случае потребуется вмешательство администратора через командную строку сервера.
